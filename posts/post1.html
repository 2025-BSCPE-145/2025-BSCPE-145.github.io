Building My First Website on GitHub Without Any Template

05 Apr 2026 • 6 min read • By Noor Fatima (BSCPE)

Starting my journey in web development, I wanted to challenge myself by building something completely on my own. Instead of relying on pre-built templates, I decided to create my first website from scratch and host it using GitHub Pages.

This project helped me understand the fundamentals of web development, including HTML structure, CSS styling, and how websites are deployed online.

Why I Chose Not to Use a Template

  • To understand how a website works internally
  • To practice writing my own code from zero
  • To improve problem-solving skills
  • To build real confidence as a developer
Building from scratch gave me a much deeper understanding of frontend development.

Technologies I Used

  • HTML5 – for structure
  • CSS3 + Flexbox – for design and responsiveness
  • GitHub Pages – for free hosting

Structure of My Website

  • index.html → Homepage
  • blog.html → All lab articles
  • /posts/ folder → Individual articles
  • css/style.css → Shared styles with dark/light mode
  • js/script.js → Theme toggle + hamburger menu

Challenges I Faced

  • Understanding correct folder structure for GitHub Pages
  • Fixing 404 errors when linking pages
  • Making the site responsive on mobile
  • Implementing dark/light mode consistently

What I Learned

  • How to create a proper multi-page website
  • Using CSS variables for theming
  • Writing clean and reusable CSS
  • Deploying and updating a live site
Tip for beginners: Always build at least one project from scratch. It teaches you way more than using templates! 🚀
