My First Website on GitHub — Built Without Any Template

By Noor Fatima

Introduction

Starting my journey in web development, I wanted to challenge myself by building something completely on my own. Instead of relying on pre-built templates, I decided to create my first website from scratch and host it using GitHub Pages.

This project helped me understand the fundamentals of web development, including HTML structure, CSS styling, and how websites are deployed online.

Why I Chose Not to Use a Template

  • To understand how a website works internally
  • To practice writing my own code
  • To improve problem-solving skills
  • To build confidence as a developer
Building from scratch gave me a deeper understanding of frontend development.

Technologies I Used

  • HTML – for structure
  • CSS – for design
  • GitHub Pages – for hosting

Structure of My Website

  • index.html → Homepage
  • blog.html → Blog listing page
  • /posts/ → Individual blog posts

Challenges I Faced

  • Understanding GitHub folder structure
  • Fixing 404 errors
  • Linking pages correctly
  • Design and layout issues

What I Learned

  • Website structure and navigation
  • Multi-page linking
  • Basic responsive design
  • How GitHub Pages works

Final Result

After completing the project, I successfully published my website online. Seeing my work live on the internet was an amazing and motivating experience.

Future Improvements

  • Add JavaScript for interactivity
  • Improve design with animations
  • Add more blog posts
  • Implement dark/light mode

Conclusion

Building my first website without using any template was challenging but rewarding. It helped me build a strong foundation in web development and boosted my confidence.

Tip: Every beginner should build at least one project from scratch 🚀
