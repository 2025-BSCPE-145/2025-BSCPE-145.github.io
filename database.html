🔹 Database Normalization (with Example)

Normalization reduces data redundancy and improves integrity. Below is a practical example using a Student-Course scenario.

Unnormalized Table (Redundant & Problematic)

StudentIDNameCoursesInstructor
101Ali KhanMath, PhysicsDr. Smith
102Sara AhmedHistoryDr. Jones

After Normalization (3NF)

Students Table
101Ali Khan
102Sara Ahmed
Courses Table
C01MathDr. Smith
C02HistoryDr. Jones
Enrollments Table
101C01
101C03
102C02

Result: No redundancy, easy updates, and better data integrity.

🔹 Database Indexes (Performance Optimization)

Indexes speed up data retrieval operations (especially SELECT queries with WHERE, ORDER BY, JOIN) at the cost of slower INSERT/UPDATE.

Common Index Types

  • Single Column Index – On frequently searched columns (e.g., StudentID)
  • Composite Index – On multiple columns used together in queries
  • Unique Index – Enforces uniqueness (e.g., Email, Roll Number)

MySQL Examples

CREATE INDEX idx_student_name ON students(name);
CREATE INDEX idx_cgpa_dept ON students(cgpa, department);
CREATE UNIQUE INDEX idx_roll_no ON students(roll_number);
CREATE INDEX idx_loan_date ON loans(issue_date);

Tip: Always analyze queries with EXPLAIN before and after adding indexes.
